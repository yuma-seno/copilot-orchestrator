# 要件定義スキル

サブエージェントが要件定義を行うためのスキルです。

## 概要

ユーザの指示やプロジェクトのコンテキストから、明確な要件を定義します。

## 手順

### 1. 現状分析

- 既存のコードベースを確認
- 関連するドキュメントを確認
- 技術スタックを把握

### 2. 要件の抽出

ユーザの指示から以下を整理します：

- **機能要件**: システムが提供すべき機能
- **非機能要件**: パフォーマンス、セキュリティ、保守性など
- **制約事項**: 技術的制約、ビジネス制約

### 3. 要件の整理

```markdown
## 要件定義書

### 背景・目的
[なぜこの変更が必要か]

### 機能要件
1. [要件1]: [詳細説明]
2. [要件2]: [詳細説明]

### 非機能要件
1. [要件1]: [詳細説明]

### 制約事項
1. [制約1]: [詳細説明]

### 受け入れ基準
1. [基準1]
2. [基準2]

### スコープ外
1. [対象外1]: [理由]
```

### 4. 不明点の洗い出し

要件に不明確な点がある場合、具体的な質問事項としてまとめます。
サマリに含めてオーケストレーター経由でユーザに確認を依頼します。

## 成果物

- `artifacts/requirements.md` - 要件定義書
- セッションディレクトリの `logs/` にタスクログを出力
