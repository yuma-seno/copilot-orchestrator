# レビュースキル

サブエージェントがコードレビューを行うためのスキルです。

## 概要

実装されたコードをレビューし、品質・セキュリティ・保守性の観点から問題点を指摘します。

## 手順

### 1. レビュー準備

- セッションディレクトリ内の要件定義書と設計書を確認
- 変更されたファイルの一覧を確認
- レビュー観点を整理

### 2. コードレビューの実施

以下の観点でレビューを行います：

#### コード品質

- コーディング規約に準拠しているか
- 命名が適切か
- 関数やクラスの責務が明確か
- 重複コードがないか
- 不要なコードが含まれていないか

#### ロジック

- 要件を正しく実装しているか
- エッジケースが考慮されているか
- エラーハンドリングが適切か
- パフォーマンスの問題がないか

#### セキュリティ

- 入力値のバリデーションが行われているか
- セキュリティ上の脆弱性がないか
- 機密情報がハードコードされていないか

#### 保守性

- コードが理解しやすいか
- テストが十分か
- ドキュメントが更新されているか
- 将来の変更に対して柔軟か

### 3. レビューレポートの作成

```markdown
## レビューレポート

### サマリ
- レビュー対象: [ファイルリスト]
- 総合評価: [承認/要修正/要再設計]
- 指摘数: 重大: [数], 軽微: [数], 提案: [数]

### 指摘事項

#### 重大（修正必須）
1. **[ファイル名:行番号]** - [指摘内容]
   - 理由: [なぜ問題か]
   - 修正案: [推奨する修正方法]

#### 軽微（修正推奨）
1. **[ファイル名:行番号]** - [指摘内容]

#### 提案（検討推奨）
1. **[ファイル名:行番号]** - [提案内容]

### 良い点
- [良い実装や設計について言及]
```

## 成果物

- `artifacts/review-report.md` - レビューレポート
- セッションディレクトリの `logs/` にレビューログを出力
