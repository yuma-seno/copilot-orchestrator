# テストスキル

サブエージェントがテスト作業を行うためのスキルです。

## 概要

実装されたコードに対してテストを作成・実行し、品質を確認します。

## 手順

### 1. テスト計画

要件と設計を確認し、テスト計画を作成します：

- **テスト対象**: 何をテストするか
- **テスト種別**: 単体テスト、結合テスト、E2Eテストなど
- **テストケース**: 正常系、異常系、境界値

### 2. テストの作成

プロジェクトの既存テストフレームワークに従ってテストを作成します：

- 既存のテストのスタイルに合わせる
- テストの目的がわかる名前をつける
- テストデータは適切に準備する

### 3. テストの実行

- 新規テストを実行して通過することを確認
- 既存テストも実行してリグレッションがないことを確認
- テスト結果をログに記録

### 4. テスト結果の記録

```markdown
## テスト結果

### テストサマリ
- 総テスト数: [数]
- 成功: [数]
- 失敗: [数]
- スキップ: [数]

### テストケース詳細
| テスト名 | 種別 | 結果 | 備考 |
|---------|------|------|------|
| [名前] | [種別] | ✅/❌ | [備考] |

### カバレッジ
- [カバレッジ情報があれば記載]

### 検出された問題
- [問題1]: [詳細と影響範囲]
```

## テスト観点

- **正常系**: 期待通りの入力で期待通りの結果が得られるか
- **異常系**: エラー入力でも適切にハンドリングされるか
- **境界値**: 境界条件で正しく動作するか
- **リグレッション**: 既存機能が壊れていないか

## 成果物

- テストコード
- セッションディレクトリの `logs/` にテスト結果ログを出力
