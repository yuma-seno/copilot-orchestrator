# 設計スキル

サブエージェントが設計作業を行うためのスキルです。

## 概要

要件定義に基づいて、実装可能な設計を作成します。

## 手順

### 1. 要件の確認

- セッションディレクトリ内の `artifacts/requirements.md` を確認
- 要件定義フェーズのログを確認
- 不明点があればサマリに含めて確認依頼

### 2. 既存コードの分析

- 変更対象のコードの構造を把握
- 依存関係を確認
- 影響範囲を分析

### 3. 設計方針の決定

以下の観点から設計方針を決定します：

- **アーキテクチャ**: どのような構造にするか
- **インターフェース**: APIやモジュール間のインターフェース
- **データ構造**: 使用するデータ構造
- **エラーハンドリング**: 異常系の処理方針

### 4. 設計書の作成

```markdown
## 設計書

### 概要
[設計の概要説明]

### アーキテクチャ
[構造の説明]

### コンポーネント設計
#### [コンポーネント名]
- 責務: [責務の説明]
- インターフェース: [入出力の定義]
- 依存: [依存するコンポーネント]

### データフロー
[データの流れの説明]

### エラーハンドリング
[異常系の処理方針]

### 影響範囲
- 変更ファイル: [ファイルリスト]
- 影響する既存機能: [機能リスト]
```

## 成果物

- `artifacts/design.md` - 設計書
- セッションディレクトリの `logs/` にタスクログを出力
