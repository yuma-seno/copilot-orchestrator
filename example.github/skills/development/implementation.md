# 実装スキル

サブエージェントが実装作業を行うためのスキルです。

## 概要

設計に基づいて、コードの実装を行います。

## 手順

### 1. 設計の確認

- セッションディレクトリ内の `artifacts/design.md` を確認
- 設計フェーズのログを確認
- 実装方針を理解

### 2. 実装の実行

以下の原則に従って実装します：

- **既存のコーディング規約に従う**: プロジェクトのスタイルに合わせる
- **小さな変更を積み重ねる**: 一度に大きな変更を行わない
- **テスタビリティを意識する**: テストしやすいコードを書く
- **エラーハンドリングを忘れない**: 異常系も適切に処理する

### 3. 変更の記録

実装した内容をログに記録します：

```markdown
## 実装ログ

### 変更ファイル
- `path/to/file1` - [変更内容の概要]
- `path/to/file2` - [変更内容の概要]

### 実装のポイント
- [重要な実装判断とその理由]

### 確認事項
- [テスト時に確認すべきポイント]
```

### 4. 基本動作確認

- 構文エラーがないことを確認
- 基本的な動作パスが機能することを確認
- 明らかなバグがないことを確認

## 実装時の注意事項

- 既存のテストを壊さない
- 不要なコードを追加しない
- コメントはプロジェクトのスタイルに合わせる
- セキュリティ上の脆弱性を作り込まない

## 成果物

- 実装されたコード
- セッションディレクトリの `logs/` に実装ログを出力
