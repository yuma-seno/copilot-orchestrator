# 状態管理スキル

セッションの状態管理に関するスキルです。

## 概要

セッションの状態を適切に管理し、エージェント間で一貫したコンテキストを維持します。

## セッション状態

### 状態遷移

```
initializing → active → completing → completed
                 ↓
              failed
```

| 状態 | 説明 |
|------|------|
| initializing | セッション初期化中 |
| active | セッション実行中 |
| completing | 完了処理中 |
| completed | 正常完了 |
| failed | 異常終了 |

## session.json の管理

### 更新ルール

- セッション情報ファイル（`session.json`）は**セッションマネージャーのみ**が更新します
- サブエージェントはセッション情報を**読み取り専用**で参照します
- 更新時は必ず履歴（`history`）にイベントを追記します

### 参照方法

サブエージェントがセッションルールを確認する場合：

1. セッションディレクトリパス内の `session.json` を読み取る
2. `rules` セクションからセッションルールを取得
3. ルールに従って作業の粒度や確認頻度を調整

## コンテキストの引き継ぎ

### サブエージェント間の引き継ぎ

複数のサブエージェントが順次タスクを実行する場合：

1. 前のサブエージェントのログを `logs/` ディレクトリから読み取る
2. `artifacts/` ディレクトリの成果物を確認
3. 判断ログ（`logs/decisions/`）から重要な判断を理解
4. これらのコンテキストを踏まえて作業を継続

### オーケストレーターへのサマリ

サブエージェントからオーケストレーターへは、セッションディレクトリ内の詳細情報ではなく、判断に必要な最小限のサマリのみを返します。
