# ログ記録スキル

セッション中のログ記録に関するスキルです。

## 概要

セッションディレクトリ内でのログ記録の規約とフォーマットを定義します。

## ログの種類

### 1. タスクログ

各タスクの実行記録です。

**ファイル名**: `logs/[timestamp]-[task-name].md`

```markdown
# タスクログ: [タスク名]

## 基本情報
- タスクID: [ID]
- 開始日時: [ISO8601]
- 完了日時: [ISO8601]
- 実行者: sub-agent
- 結果: [成功/失敗]

## 実行内容
[実行した作業の詳細]

## 変更されたファイル
- [ファイルパス]: [変更概要]

## 判断記録
- [判断が必要だった場面と判断内容]

## 次のステップへの引き継ぎ
- [次のタスクで注意すべき事項]
```

### 2. 判断ログ

重要な判断の記録です。

**ファイル名**: `logs/decisions/[timestamp]-[decision-name].md`

```markdown
# 判断記録: [判断の概要]

## 状況
[判断が必要になった背景]

## 選択肢
1. [選択肢A]: [メリット/デメリット]
2. [選択肢B]: [メリット/デメリット]

## 判断
[選択した選択肢と理由]

## 影響
[この判断による影響]
```

## ログ記録のルール

1. **タイムスタンプ**: ファイル名にタイムスタンプを含める（`yyyymmddhhmmss` 形式）
2. **簡潔さ**: 事実を簡潔に記録する
3. **トレーサビリティ**: 後から経緯を追跡できるように記録する
4. **機密情報**: パスワードやAPIキーなどの機密情報はログに含めない
