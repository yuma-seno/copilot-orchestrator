# タスク移譲スキル

サブエージェントへのタスク移譲の方法と指示フォーマットを定義します。

## 移譲時の指示テンプレート

サブエージェントに作業を移譲する際は、以下の情報を**必ず**含めてください：

```
@sub-agent 以下のタスクを実行してください：

## タスク情報
- **タスク名**: [簡潔なタスク名（例: 「認証APIの実装」）]
- **タスクID**: [task-001 形式の一意な識別子]
- **目的**: [何を達成するか（1〜2文で）]

## 責任範囲
- **対象ファイル/ディレクトリ**: [具体的なパス（例: src/auth/）]
- **対象外**: [変更してはいけないファイルや機能]

## セッション情報
- **セッションディレクトリ**: [.copilot/orchestrator/[sessionId]/]
- **参照すべきログ**: [前のタスクのログパスがあれば]
- **参照すべき成果物**: [要件定義書や設計書のパスがあれば]

## 作業指示
- **参照スキル**: [参照すべきスキルファイルのパス]
- **具体的な作業内容**: [実行すべき作業を箇条書きで]

## 完了条件
- [条件1（例: 「全テストが通過すること」）]
- [条件2（例: 「既存のAPIインターフェースが維持されること」）]

## 品質基準
- [基準1（例: 「TypeScriptの型エラーがないこと」）]
- [基準2（例: 「ESLintの警告がないこと」）]

## セッション情報の更新
- タスク完了後、session.json の tasks に以下を追記してください：
  {"id": "[taskId]", "name": "[タスク名]", "status": "completed", "assignedAt": "[ISO8601]", "completedAt": "[ISO8601]", "summary": "[サマリ]"}
- history にも完了イベントを追記してください
```

## 指示の粒度ガイドライン

### 明確にすべきこと（MUST）

- **何を達成するか**（目的と完了条件）
- **どこが対象か**（ファイルパスやモジュール）
- **どこにログを出すか**（セッションディレクトリパス）
- **何を参照すべきか**（スキルファイル、過去のログ）

### 指示しないこと（MUST NOT）

- 具体的な実装方法（サブエージェントがスキルと判断で決める）
- 使用する関数名やクラス名（サブエージェントが設計する）
- コードの書き方の詳細（サブエージェントがコーディング規約に従う）

### 状況に応じて指示すること（OPTIONAL）

- 参考にすべき既存実装（「src/auth/login.ts を参考に」）
- 優先すべき観点（「パフォーマンスを重視」「セキュリティを最優先」）
- 注意すべき制約（「外部APIのレスポンス形式は変更不可」）

## サマリの受け取りと判断

### 正常完了の場合

サブエージェントから以下の形式でサマリが返されます：

```markdown
## 完了サマリ
- タスク: [タスク名]
- 結果: 成功
- 変更概要: [1〜3行の概要]
- 注意事項: [あれば]
- 詳細ログ: [ログパス]
```

**次のアクション：**
1. `skills/orchestration/quality-gate.md` を参照して品質基準を確認
2. 基準を満たしていれば `todo` でタスクを完了にする
3. 次のタスクに移行する

### 失敗の場合

```markdown
## 完了サマリ
- タスク: [タスク名]
- 結果: 失敗
- 失敗理由: [理由]
- 詳細ログ: [ログパス]
```

**リトライ判断基準：**

| 失敗の種類 | アクション |
|-----------|-----------|
| 一時的なエラー（ネットワーク等） | 同じ指示でリトライ |
| 指示の不足・不明確 | 指示を具体化してリトライ |
| 技術的な困難 | ユーザに状況を報告して判断を仰ぐ |
| スコープ外の問題 | ユーザに報告してスコープの再検討 |

**リトライ時の追加情報：**
- 前回の失敗ログのパスを伝える
- 失敗原因に対する追加のコンテキストや制約を伝える
- 必要に応じてスコープや完了条件を調整する

### 要確認の場合

```markdown
## 完了サマリ
- タスク: [タスク名]
- 結果: 要確認
- 確認事項: [ユーザに確認すべき内容]
- 詳細ログ: [ログパス]
```

**次のアクション：**
1. `askQuestion` でユーザに確認事項を伝える
2. ユーザの回答に基づいて次の指示を決定する
