# セッション初期化スキル

オーケストレーションセッションの初期化手順を定義します。

## セッションディレクトリ構造

以下のディレクトリ構造を作成します：

```
.copilot/orchestrator/[yyyymmddhhmmssfff]/
├── session.json          # セッション情報ファイル（状態、ルール、タスク、履歴）
├── summary.md            # セッション完了時の最終サマリ
├── logs/                 # タスク実行ログ
│   └── decisions/        # 重要な判断の記録
└── artifacts/            # 成果物（要件定義書、設計書、レビューレポート等）
```

## セッションIDの生成ルール

- 形式: `yyyymmddhhmmssfff`
  - `yyyy`: 西暦4桁
  - `mm`: 月2桁（01-12）
  - `dd`: 日2桁（01-31）
  - `hh`: 時2桁（00-23）
  - `mm`: 分2桁（00-59）
  - `ss`: 秒2桁（00-59）
  - `fff`: ミリ秒3桁（000-999）
- 例: `20260329123456789` → 2026年3月29日 12:34:56.789

## session.json の初期構造

```json
{
  "sessionId": "[yyyymmddhhmmssfff]",
  "createdAt": "[ISO8601形式の日時]",
  "status": "active",
  "rules": {
    "questionGranularity": null,
    "reviewPolicy": null,
    "scope": null
  },
  "tasks": [],
  "history": [
    {
      "timestamp": "[ISO8601]",
      "event": "session-init",
      "details": "セッション初期化完了"
    }
  ]
}
```

## セッションルールの定義

### 質問粒度（questionGranularity）

ユーザに `askQuestion` で確認する頻度を制御します：

| 値 | 動作 | 適用場面 |
|----|------|---------|
| `high` | 各タスク開始前・完了後に確認。重要な判断は都度確認。 | 初めてのプロジェクト、重要な変更 |
| `medium` | フェーズの切り替え時に確認。判断は問題がある場合のみ確認。 | 通常の開発作業 |
| `low` | 必須の確認のみ。判断はサブエージェントに任せる。 | 定型的な作業、信頼度の高い作業 |

### レビュー方針（reviewPolicy）

各フェーズ完了時のレビュー方法を制御します：

| 値 | 動作 |
|----|------|
| `auto` | サブエージェントが自動的にレビューし、問題があれば修正 |
| `manual` | ユーザがレビュー。サブエージェントはレポートのみ出力 |
| `skip` | レビューフェーズをスキップ |

### スコープ（scope）

作業対象の範囲を記述します：
- ファイルパターン（例: `src/components/**`）
- 機能範囲（例: 「ユーザ認証機能」）
- 制約事項（例: 「既存のAPIインターフェースは変更しない」）

## 初期化手順

### 手順1: セッションディレクトリの作成

`@sub-agent` に以下を指示します：

```
セッションディレクトリを作成してください：
1. .copilot/orchestrator/[sessionId]/ ディレクトリを作成
2. logs/ サブディレクトリを作成
3. logs/decisions/ サブディレクトリを作成
4. artifacts/ サブディレクトリを作成
5. session.json を初期構造で作成
6. summary.md を空ファイルで作成
```

### 手順2: セッションルールの確認

`askQuestion` でユーザに以下を確認します：

```
オーケストレーションセッションを開始します。以下のルールを確認させてください：

1. **質問粒度**: 作業中にどの程度の粒度で確認を求めますか？
   - 高: 各タスクの開始前・完了後に都度確認
   - 中: フェーズの切り替え時に確認（推奨）
   - 低: 最小限の確認のみ

2. **レビュー方針**: 各フェーズ完了時のレビュー方針は？
   - 自動: サブエージェントが自動レビュー
   - 手動: ユーザが確認
   - スキップ: レビューなし

3. **スコープ**: 作業対象の範囲を教えてください。
   （ファイルパス、機能名、制約事項など）
```

### 手順3: セッションルールの記録

`@sub-agent` に `session.json` の `rules` フィールドを更新させます：

```json
{
  "rules": {
    "questionGranularity": "[high|medium|low]",
    "reviewPolicy": "[auto|manual|skip]",
    "scope": "[ユーザが指定したスコープ]"
  }
}
```

同時に `history` にルール設定イベントを追記：

```json
{
  "timestamp": "[ISO8601]",
  "event": "rules-set",
  "details": "セッションルール設定完了: 質問粒度=[値], レビュー方針=[値]"
}
```

## セッション完了手順

セッション完了時に `@sub-agent` に以下を指示します：

1. `session.json` の `status` を `"completed"` に更新
2. `history` に完了イベントを追記
3. `summary.md` に最終サマリを記載（全タスクの結果一覧、変更ファイル一覧、特記事項）
