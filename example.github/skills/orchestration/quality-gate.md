# 品質ゲートスキル

オーケストレーターがフェーズ間の品質を判定するためのスキルです。

## 概要

各フェーズの完了時に品質基準を満たしているかを判定し、次のフェーズに進むか差し戻すかを決定します。

## 品質ゲート定義

### 要件定義ゲート

次フェーズ（設計）に進むための条件：

- [ ] 要件が具体的かつ曖昧さがないこと
- [ ] ユーザの確認が取れていること
- [ ] スコープが明確であること
- [ ] 受け入れ基準が定義されていること

### 設計ゲート

次フェーズ（実装）に進むための条件：

- [ ] 技術的な実現可能性が確認されていること
- [ ] 既存コードへの影響が分析されていること
- [ ] 設計方針が文書化されていること
- [ ] エッジケースが考慮されていること

### 実装ゲート

次フェーズ（テスト）に進むための条件：

- [ ] コードが完成していること
- [ ] 基本的な動作確認が行われていること
- [ ] コーディング規約に準拠していること
- [ ] 不要なコードが含まれていないこと

### テストゲート

次フェーズ（レビュー）に進むための条件：

- [ ] 全テストが通過していること
- [ ] エッジケースがテストされていること
- [ ] テストカバレッジが基準を満たしていること

### レビューゲート

完了のための条件：

- [ ] コード品質の問題が解消されていること
- [ ] セキュリティ上の懸念がないこと
- [ ] パフォーマンスの問題がないこと
- [ ] ドキュメントが更新されていること

## 品質ゲート判定フロー

```
サブエージェントの完了サマリを受信
    ↓
品質ゲートチェックリストを確認
    ↓
  ┌─ 全項目クリア → 次のフェーズへ進む
  │
  ├─ 軽微な問題あり → サブエージェントに修正を指示して再チェック
  │
  └─ 重大な問題あり → 前のフェーズに差し戻し
```

## 品質レベル

セッションルールの質問粒度に応じて品質チェックの厳密さを調整します：

| 質問粒度 | 品質チェック |
|---------|------------|
| 高 | 全ゲートの全項目を厳密にチェック。各項目でユーザ確認を求める |
| 中 | 重要項目をチェック。問題がある場合のみユーザ確認 |
| 低 | 最低限のチェックのみ。重大な問題のみ報告 |
