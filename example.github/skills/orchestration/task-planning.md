# タスク計画スキル

オーケストレーターがタスクを分解・計画するためのスキルです。

## 概要

ユーザからの指示を分析し、実行可能なタスクに分解して計画を立てます。

## タスク分解の手順

### 1. 指示の分析

ユーザの指示から以下を抽出します：

- **目的**: 最終的に何を達成したいか
- **成果物**: どのような成果物が必要か
- **制約**: 時間、技術、スコープの制約
- **依存関係**: 外部サービスや既存コードへの依存

### 2. タスクの分解

指示を以下の粒度でタスクに分解します：

1. **フェーズレベル**: 要件定義 → 設計 → 実装 → テスト → レビュー
2. **機能レベル**: フェーズ内の機能単位の作業
3. **タスクレベル**: 機能内の具体的な作業項目

### 3. 依存関係の整理

タスク間の依存関係を明確にします：

- **直列タスク**: 前のタスクが完了しないと開始できないもの
- **並列タスク**: 同時に実行可能なもの
- **条件付きタスク**: 特定の条件で実行するもの

### 4. 優先度の決定

各タスクに優先度を設定します：

| 優先度 | 基準 |
|--------|------|
| 高 | 他のタスクがブロックされる、クリティカルパス上にある |
| 中 | 重要だが他タスクをブロックしない |
| 低 | あれば望ましいが必須ではない |

## タスク計画テンプレート

```markdown
## タスク計画

### 目的
[達成したいこと]

### フェーズ1: [フェーズ名]
- [ ] タスク1-1: [内容] (優先度: 高, 依存: なし)
- [ ] タスク1-2: [内容] (優先度: 高, 依存: タスク1-1)

### フェーズ2: [フェーズ名]
- [ ] タスク2-1: [内容] (優先度: 中, 依存: フェーズ1完了)

### 完了条件
- [条件1]
- [条件2]
```

## 注意事項

- タスクは小さすぎず大きすぎない適切な粒度にします
- 各タスクには明確な完了条件を設定します
- 不確実な部分は調査タスクとして分離します
