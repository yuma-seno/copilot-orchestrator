# copilot-orchestrator

VSCode GitHub Copilot向けのエージェントオーケストレーションシステム

## 概要

このリポジトリは、VSCode の GitHub Copilot で利用するためのエージェントオーケストレーションシステムのサンプル実装です。3つのエージェントが連携して開発タスクを効率的に遂行します。

## アーキテクチャ

```
ユーザ
  ↕ (askQuestion)
オーケストレーター
  ↕ (agent)                    ↕ (agent)
サブエージェント            セッションマネージャー
  ↕ (skills参照)              ↕ (セッションディレクトリ管理)
ファイルシステム             .copilot/orchestrator/[session-id]/
```

### エージェント構成

| エージェント | 役割 | ツール |
|-------------|------|--------|
| **オーケストレーター** | 全体の流れを管理し、サブエージェントに作業を移譲 | askQuestion, todo, agent |
| **サブエージェント** | 実際の開発作業（要件定義〜レビュー）を実行 | 全ツール |
| **セッションマネージャー** | セッションのライフサイクル管理 | ファイル操作 |

### オーケストレーションフロー

```
1. セッション初期化（セッションマネージャー）
2. セッションルール確認（ユーザ）
3. タスク分析・計画（オーケストレーター）
4. 作業実行（サブエージェント）
5. 品質ゲート判定（オーケストレーター）
6. セッション完了（セッションマネージャー）
```

## セッションディレクトリ

オーケストレーションの実行時に、以下のディレクトリ構造が自動で作成されます：

```
.copilot/orchestrator/[yyyymmddhhmmssfff]/
├── session.json          # セッション情報（ルール、タスク、履歴）
├── summary.md            # セッションサマリ
├── logs/                 # タスク実行ログ
│   └── decisions/        # 判断記録
└── artifacts/            # 成果物（要件定義書、設計書、レビューレポート等）
```

## 使い方

### 1. セットアップ

`example.github/` フォルダの内容をプロジェクトの `.github/` フォルダにコピーします：

```bash
cp -r example.github/agents/ .github/agents/
cp -r example.github/skills/ .github/skills/
```

### 2. オーケストレーターの起動

VSCode の GitHub Copilot Chat で、オーケストレーターエージェントを呼び出します：

```
@orchestrator [開発タスクの指示]
```

### 3. セッションルールの確認

オーケストレーターが以下のルールを確認します：

- **質問粒度**: 高（都度確認）/ 中（要所で確認）/ 低（最小限の確認）
- **レビュー方針**: 自動 / 手動 / スキップ
- **スコープ**: 作業対象の範囲

### 4. 作業の実行

オーケストレーターがタスクを計画し、サブエージェントに移譲して作業を進めます。
進捗はセッションディレクトリ内のログで確認できます。

## 設計思想

### オーケストレーターの役割分離

オーケストレーターは「何をすべきか」を判断し、「どうやるか」はサブエージェントに委ねます。これにより：

- オーケストレーターのコンテキストウィンドウを節約
- サブエージェントが専門的な作業に集中可能
- セッションディレクトリを介した情報共有で、エージェント間の連携を実現

### セッションディレクトリによるコンテキスト管理

サブエージェントの詳細な出力はセッションディレクトリ内にファイルとして記録され、オーケストレーターには最小限のサマリのみが返されます。これにより：

- エージェント間の情報ロスを防止
- 作業の再現性と追跡可能性を確保
- 複数のサブエージェント間でのコンテキスト引き継ぎを実現

### スキルによる作業手順の標準化

具体的な作業手順はスキルファイルに定義されており、サブエージェントはこれを参照して作業を進めます。これにより：

- 作業品質の均一化
- 手順の改善が容易
- エージェントへの指示を最小限に抑制
